--- title: Units ---

Until now we have been using examples with values like 50 and not specifying units. Our examples were Unitless and that is just fine for abstract models. When you need to specify the units you can use makerjs.unitType which has these properties:

  • Centimeter
  • Foot
  • Inch
  • Meter
  • Millimeter

These properties are case sensitive. They contain the values "cm", "foot", "inch", "m" and "mm" respectively. It is your choice whether to use the named property or the string value directly.

Units and SVG rendering

SVG on the screen is displayed in a virtual viewbox so that you may see the whole drawing, but in the SVG markup itself the units are preserved. Let's change our units to Foot and see how they render on screen:

{% highlight javascript %} var makerjs = require('makerjs'); function myModel() { var line = { id: 'myLineId', type: 'line', origin: [0, 0], end: [1, 1] }; var circle = { id: 'myCircleId', type: 'circle', origin: [0, 0], radius: 1 }; var pathArray = [ line, circle ]; //set properties using the "this" keyword this.id = 'myModel'; this.paths = pathArray; } var model1 = new myModel(); var model2 = new myModel(); //they will be on top of each other, so let's move the origin model2.origin = [2, 0]; var model = { units: makerjs.unitType.Foot, id: 'iHave2models', models: [ model1, model2 ] }; var svg = makerjs.exporter.toSVG(model); document.write(svg); {% endhighlight %}
