--- title: Model tree structures ---

A model is a tree structure which may contain paths, and it may also contain other models in a heirachy. Let's look at an example:

{% highlight javascript %} //mounting plate var makerjs = require('makerjs'); var plate = { models: { outer: makerjs.model.center(new makerjs.models.RoundRectangle(120, 100, 10)), bolts: makerjs.model.center(new makerjs.models.BoltRectangle(100, 80, 5)) }, paths: { hole: new makerjs.paths.Circle(25) } } var svg = makerjs.exporter.toSVG(plate); document.write(svg); {% endhighlight %}

If we represent this drawing as a conceptual tree structure, it would look like this: 

plate
+-models
| +-outer
| | +-paths
| |   +-TopLeft
| |   +-Top
| |   +-TopRight
| |   +-Left
| |   +-Right
| |   +-BottomLeft
| |   +-Bottom
| |   +-BottomRight
| +-bolts
|   +-paths
|     +-TopLeft
|     +-TopRight
|     +-BottomLeft
|     +-BottomRight
+paths
 +-hole

(You may notice that this structure is reminiscent of a folder system on your computer.) We can easily traverse the tree when starting at the root. For example, let's change the radius of the BottomRight bolt hole:

{% highlight javascript %} //mounting plate var makerjs = require('makerjs'); var plate = { models: { outer: makerjs.model.center(new makerjs.models.RoundRectangle(120, 100, 10)), bolts: makerjs.model.center(new makerjs.models.BoltRectangle(100, 80, 5)) }, paths: { hole: new makerjs.paths.Circle(25) } } //change radius of BottomRight bolt hole plate.models.bolts.paths.BottomRight.radius = 2; var svg = makerjs.exporter.toSVG(plate); document.write(svg); {% endhighlight %}

We can access the BottomRight circle from the plate downward. Notice that in this tree structure, you cannot access upwardly. The plate object contains BottomRight and its parents, but BottomRight does not have a relationship with its container. To illustrate this, let's create a local valiable for BottomRight:

{% highlight javascript %} //mounting plate var makerjs = require('makerjs'); var plate = { models: { outer: makerjs.model.center(new makerjs.models.RoundRectangle(120, 100, 10)), bolts: makerjs.model.center(new makerjs.models.BoltRectangle(100, 80, 5)) }, paths: { hole: new makerjs.paths.Circle(25) } } //create a local variable for BottomRight bolt hole var bottomRight = plate.models.bolts.paths.BottomRight; bottomRight.radius = 2; //bottomRight is a circle - it only has radius and origin properties. var svg = makerjs.exporter.toSVG(plate); document.write(svg); {% endhighlight %}

There are no properties of the bottomRight circle object which access anything up the tree. It also does not have any references to its sibling circles within bolt.paths. This is the nature of a simple tree structure.
