--- title: Routes ---

Given the following example tree structure, we are able to refer to deep objects by using the dot notation: 

plate
+-models
| +-outer
| | +-paths
| |   +-TopLeft
| |   +-Top
| |   +-TopRight
| |   +-Left
| |   +-Right
| |   +-BottomLeft
| |   +-Bottom
| |   +-BottomRight
| +-bolts
|   +-paths
|     +-TopLeft_bolt
|     +-TopRight_bolt
|     +-BottomLeft_bolt
|     +-BottomRight_bolt
+paths
 +-hole
{% highlight javascript %} var bottomRight_bolt = plate.models.bolts.paths.BottomRight_bolt; {% endhighlight %}

Every object in the tree is accessible via a route like this. Let's expand all possible routes of this tree that reach a child model or a child path - and since we know they each will begin with plate, we will omit the root object from each result:

{% highlight javascript %} models.outer models.outer.paths.TopLeft models.outer.paths.Top models.outer.paths.TopRight models.outer.paths.Left models.outer.paths.Right models.outer.paths.BottomLeft models.outer.paths.Bottom models.outer.paths.BottomRight models.bolts models.bolts.paths.TopLeft_bolt models.bolts.paths.TopRight_bolt models.bolts.paths.BottomLeft_bolt models.bolts.paths.BottomRight_bolt paths.hole {% endhighlight %}

Instead of the dot notation, let's put each segment into an array element:

{% highlight javascript %} ["models", "outer"] ["models", "outer", "paths", "TopLeft"] ["models", "outer", "paths", "Top"] ["models", "outer", "paths", "TopRight"] ["models", "outer", "paths", "Left"] ["models", "outer", "paths", "Right"] ["models", "outer", "paths", "BottomLeft"] ["models", "outer", "paths", "Bottom"] ["models", "outer", "paths", "BottomRight"] ["models", "bolts"] ["models", "bolts", "paths", "TopLeft_bolt"] ["models", "bolts", "paths", "TopRight_bolt"] ["models", "bolts", "paths", "BottomLeft_bolt"] ["models", "bolts", "paths", "BottomRight_bolt"] ["paths", "hole"] {% endhighlight %}

These are what is known in Maker.js as a route. It is simply an array of strings that represent the segments names to traverse a tree.

Additionally, we can "flatten" a route array into a string, known as a route key, by calling makerjs.createRouteKey(route) and passing a route. Every route key is of course unique in the scope of the root object. You may use it as a way of identifying a child path or model.
