--- title: Path constructors ---

In the example code above we used plain old JavaScript objects to create paths and models. Notice that we didn't need to use a special constructor provided by Maker.js to create either a path or a model. This is an intentional aspect of Maker.js, that you can decide how to create your objects. To make these plain objects work with Maker.js, they needed to use the property names specified above.

We also illustrated 3 ways of defining an object: using a var, using a function that returns a var, and using a constructor function (for use by the new keyword). Let's revisit our simple line path example, and convert it to a constructor function.

{% highlight javascript %} var makerjs = require('makerjs'); var line = { id: 'myLineId', type: 'line', origin: [0, 0], end: [50, 50] }; var svg = makerjs.exporter.toSVG(line); document.write(svg); {% endhighlight %} try it in JsFiddle ⇗ {% highlight javascript %} var makerjs = require('makerjs'); function line() { this.id = 'myLineId', this.type = 'line', this.origin = [0, 0], this.end = [50, 50] } var svg = makerjs.exporter.toSVG(new line()); document.write(svg); {% endhighlight %} try it in JsFiddle ⇗

Of course this example is not very useful because it only produces a line with the same id, origin and end every time. Instead, these should be passed as parameters.

Since this is a common scenario, Maker.js provides constructors for all primitive paths: line, circle and arc:

  • makerjs.paths.Line(id: string, origin: point, end: point)
  • makerjs.paths.Circle(id: string, origin: point, radius: number)
  • makerjs.paths.Arc(id: string, origin: point, radius: number, startAngle: number, endAngle: number)
{% highlight javascript %} var makerjs = require('makerjs'); var line = new makerjs.paths.Line('myLineId', [0, 0], [50, 50]); var circle = new makerjs.paths.Circle('myCircleId', [0, 0], 50); var arc = new makerjs.paths.Arc('myLineId', [0, 0], 25, 0, 90); var svg = makerjs.exporter.toSVG([line, circle, arc]); document.write(svg); {% endhighlight %} try it in JsFiddle ⇗
