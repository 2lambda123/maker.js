--- title: Bezier curves ---

Bezier curves are a fascinating and complex topic too large to cover here, it is recommended that you visit A Primer on Bezier Curves by Mike Kamermans (aka Pomax). Maker.js depends on Pomax's Bezier.js which is a vital site to visit for understanding Bezier curve functionality in depth.

Bezier curves: from wild to mild

The central idea of Maker.js is to break a drawing up into small, reasonable primitives such as lines circles and arcs. These primitives have a monotonic nature which make them easy to understand when it comes to bounding box and intersection calculation. Bezier Curves may make these operations more difficult, since they can potentially represent some "wild" shapes such as loops, sharp corners, or S shapes that are not monotonic in both x and y dimensions. But they can also represent "mild" shapes which resemble circular arcs, with a monotonic shape that is more predictable.

The representation of a Bezier curve in Maker.js is a model containing a series of "mild" curves which are divisions of the original curve.

To create a Bezier Curve, call makerjs.models.BezierCurve with the new operator. You can create both quadratic and cubic Bezier curves.

Create a quadratic Bezier curve in by passing an array of three points - an origin point, a control point, and an end point:

{% highlight javascript %} //create a quadratic Bezier curve var m = require('makerjs'); var points = [[0, 100], [0, 0], [100, 0]]; var curve1 = new m.models.BezierCurve(points); curve1.origin = [20, 20]; //more accurate var curve2 = new m.models.BezierCurve(points, 0.01); var model = { models: { c1: curve1, c2: curve2 } }; var svg = m.exporter.toSVG(model); document.write(svg); {% endhighlight %}

Create a cubic Bezier curve in by passing an array of four points - an origin point, a first control point (relating to the origin point), a second control point (relating to the end point), and an end point:

{% highlight javascript %} //create a cubic Bezier curve var m = require('makerjs'); var points = [[0, 0], [50, 50], [50, -50], [100, 0]]; var curve1 = new m.models.BezierCurve(points); curve1.origin = [0, 20]; //more accurate var curve2 = new m.models.BezierCurve(points, 0.1); var model = { models: { c1: curve1, c2: curve2 } }; var svg = m.exporter.toSVG(model); document.write(svg); {% endhighlight %}

Play these examples, then click "show path names" to see the "mild" curves representing the overall curve.

Bezier Curves represented with circular arcs

In Maker.js version previous of 0.18.0, Bezier Curves were represented as a model containing a series of circular arc paths which closely approximate the curve.

To achieve the previous behavior, Call makerjs.models.BezierCurveAsArcs with the new operator. You can create both quadratic and cubic Bezier curves. For either type, you may optionally pass the accuracy - the maximum distance between the true curve and the arc approximations. Closer approximation means more calculation time and more arcs. The default accuracy coefficient in Maker.js will produce an accurate and visually smooth curve in a reasonable calculation timeframe.

Create a quadratic Bezier curve in by passing an array of three points - an origin point, a control point, and an end point:

{% highlight javascript %} //create a quadratic Bezier curve var m = require('makerjs'); var points = [[0, 100], [0, 0], [100, 0]]; var curve1 = new m.models.BezierCurveAsArcs(points); curve1.origin = [20, 20]; //more accurate var curve2 = new m.models.BezierCurveAsArcs(points, 0.01); var model = { models: { c1: curve1, c2: curve2 } }; var svg = m.exporter.toSVG(model); document.write(svg); {% endhighlight %}

Create a cubic Bezier curve in by passing an array of four points - an origin point, a first control point (relating to the origin point), a second control point (relating to the end point), and an end point:

{% highlight javascript %} //create a cubic Bezier curve var m = require('makerjs'); var points = [[0, 0], [50, 50], [50, -50], [100, 0]]; var curve1 = new m.models.BezierCurveAsArcs(points); curve1.origin = [0, 20]; //more accurate var curve2 = new m.models.BezierCurveAsArcs(points, 0.1); var model = { models: { c1: curve1, c2: curve2 } }; var svg = m.exporter.toSVG(model); document.write(svg); {% endhighlight %}

Play these examples, then click "show path names" to see the arcs representing the curve.
